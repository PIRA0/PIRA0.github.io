REST API Research Project

Evoluzione delle architetture API e dashboard videogiochi

Le API moderne sono il risultato di una lunga evoluzione tecnologica che ha trasformato il modo in cui i sistemi comunicano tra loro. In origine, le applicazioni utilizzavano RPC (Remote Procedure Call), un approccio diretto ma fortemente dipendente dall’implementazione. Successivamente SOAP introdusse uno standard più rigoroso basato su XML, garantendo interoperabilità ma al costo di maggiore complessità e pesantezza.

Con l’aumento delle applicazioni web e mobile, è emersa la necessità di un approccio più leggero e scalabile. REST (Representational State Transfer) ha risposto perfettamente a questa esigenza, sfruttando HTTP, JSON e un’architettura stateless. Questo ha permesso una separazione netta tra client e server, facilitando lo sviluppo di API pubbliche, microservizi e sistemi cloud-native.

Oggi REST domina il panorama grazie alla sua semplicità, velocità e compatibilità universale con browser, app mobile e sistemi distribuiti.

Documentazione Tecnica REST API

Endpoint utilizzati

L’applicazione utilizza la API pubblica RAWG per ottenere dati sui videogiochi. Tutti gli endpoint rispettano le best practice REST:

GET /games?key=API_KEY&page_size=9
GET /games?search={query}&key=API_KEY
GET /games/{id}

Gli endpoint sono basati su risorse nominali (games) e non su verbi, seguendo il paradigma REST.

Metodi e Payload JSON

Le risposte della API sono in formato JSON e vengono gestite lato client tramite JavaScript.

{ "id": 3498, "name": "Grand Theft Auto V", "released": "2013-09-17", "rating": 4.5, "genres": [ { "name": "Action" }, { "name": "Adventure" } ] }

Il client estrae dinamicamente nome, rating, data di uscita e generi per costruire le card.

Gestione errori

404 Not Found

La risorsa richiesta non esiste o la query non produce risultati.

401 Unauthorized

API key mancante o non valida.

500 Server Error

Errore interno del servizio RAWG o problema di rete.

Conclusione

Il progetto dimostra l’utilizzo pratico delle API REST in un contesto reale, evidenziando la gestione delle richieste HTTP, la manipolazione dei dati JSON e la creazione di un’interfaccia dinamica e responsive.
